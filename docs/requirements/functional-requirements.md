# 機能要件

## 1. 対象範囲
Phase1では、開発者本人を主な利用者とし、
現在の資産状況の把握と、目的達成可否の判断を支援する。

提供する主な機能は、以下とする。

- 資産口座の管理
- 月末時点の資産残高の記録
- 金融商品別の残高の記録
- CSVによる月末残高の取り込み
- 総資産、口座別資産、商品別資産の推移表示
- 手取り収入の記録
- 目的達成可否のシミュレーション

## 共通ルール
金額は日本円とし、
整数で管理する。

外貨および為替換算は、
Phase1の対象外とする。

## 2. 資産口座管理
### 2.1 資産口座・保管場所の登録
利用者は、資産を保有している口座、制度または保管場所を登録できる。
登録対象の例は以下とする。
- 家にある現金
- 財布
- 銀行口座
- 証券口座
- iDeCo
- 企業型DC

登録時には、少なくとも以下を設定できる。
- 口座・保管場所名
- 種別
- 金融機関または運営機関
- 利用可能資産として扱うか
- 残高の記録単位
- 有効または無効の状態

残高の記録単位は、以下から選択できる。
- 口座単位
- 商品単位

口座単位を選択した場合、
当該口座または保管場所の月末残高を直接登録する。

商品単位を選択した場合、
当該口座に属する保有商品の月末評価額を登録し、
その合計額を口座の月末残高として扱う。

### 2.2 資産口座・保管場所の編集
利用者は、登録済みの資産口座・保管場所情報を編集できる。
編集可能な項目は以下とする。

- 口座・保管場所名
- 種別
- 金融機関または運営機関
- 利用可能資産区分
- 残高の記録単位
- 有効状態

残高履歴が登録済みの資産口座について、
残高の記録単位は変更できるものとする。

変更する場合は、
過去の資産推移や集計結果に影響を与える可能性があるため、
変更前に確認を表示する。

変更後の集計方法および資産推移への反映方法は、
詳細設計で定義する。

### 2.3 資産口座・保管場所の無効化
過去の残高履歴を保持したまま利用を終了できるよう、
登録済みの資産口座・保管場所を削除せず無効化できる。

無効化した資産口座・保管場所は、
新しい月末残高の登録対象には表示しない。

ただし、過去の月末残高や資産推移には引き続き表示する。

### 2.4 月末残高の重複計上防止
同一の資産口座について、
口座単位の月末残高と商品単位の月末評価額を、
同じ対象年月の総資産へ重複して加算しないものとする。

残高の記録単位が商品単位の場合、
保有商品の月末評価額の合計を口座残高として扱う。

## 3. 月末資産残高管理
### 3.1 月末資産残高の登録
利用者は、登録済みの資産口座または保管場所について、
対象月の月末時点における残高を登録できる。

登録方法は以下とする。
- 手入力
- CSVインポート

登録時には、少なくとも以下を設定できる。
- 対象年月
- 資産口座・保管場所
- 月末残高
- 備考

同一の資産口座・保管場所について、
同一対象年月の月末残高を重複して登録できないものとする。

### 3.2 月末資産残高の修正
利用者は、登録済みの月末資産残高を修正できる。
修正可能な項目は以下とする。
- 対象年月
- 資産口座・保管場所
- 月末残高
- 備考

月末資産残高は、資産推移の根拠となるデータであるため、
論理削除および物理削除は行わない

### 3.3 CSVによる一括登録
利用者は、複数の月末資産残高をCSVから一括登録できる。
CSVインポートおよびCSV出力の詳細は、
「5. CSVインポート」に定義する。

## 4. 保有商品管理

### 4.1 保有商品の登録
利用者は、証券口座、iDeCo、企業型DCなどの資産口座に対して、
保有している金融商品を登録できる。

登録時には、少なくとも以下を設定できる。
- 資産口座
- 商品名・銘柄名
- 商品種別
- 備考
- 有効または無効の状態

商品種別の例は以下とする。
- 投資信託
- 国内株式
- 外国株式
- 預金商品
- その他

### 4.2 保有商品の編集
利用者は、登録済みの保有商品情報を編集できる。
編集可能な項目は以下とする。

- 資産口座
- 商品名・銘柄名
- 商品種別
- 備考
- 有効状態

### 4.3 保有商品の無効化
過去の商品別月末残高履歴を保持するため、
利用を終了または売却した保有商品は削除せず無効化できる。

無効化された保有商品は、新しい月末残高の登録対象には表示しない。
過去の資産推移には引き続き表示する。

### 4.4 商品別月末残高の登録

利用者は、登録済みの保有商品について、
対象月の月末時点における評価額を登録できる。

登録時には、少なくとも以下を設定できる。
- 対象年月
- 保有商品
- 月末評価額
- 備考

同一の保有商品について、
同一対象年月の月末評価額を重複して登録できないものとする。

### 4.5 商品別月末残高の修正
利用者は、登録済みの商品別月末評価額を修正できる。

商品別月末評価額は、
資産推移の根拠となるデータであるため、
論理削除および物理削除は行わない。

### 4.6 資産口座残高への集計
資産口座ごとに、残高の記録方法を以下から選択できる。
- 口座単位で月末残高を記録する
- 商品単位で月末評価額を記録する

商品単位で記録する資産口座については、
その口座に属する商品別月末評価額の合計を、
当該資産口座の月末残高として扱う。

口座単位の月末残高と商品別月末評価額を
重複して総資産へ加算しないものとする。


## 5. CSVインポート
### 5.1 インポート対象
利用者は、月末時点の資産残高をCSVファイルから一括登録できる。
Phase1では、以下のCSVインポートに対応する。

- 口座単位の月末資産残高
- 商品単位の商品別月末評価額

口座単位の月末資産残高と商品単位の商品別月末評価額は、
異なるCSVファイルとして取り扱う。

### 5.2 月末資産残高CSV
口座単位で残高を管理する資産口座・保管場所について、
月末資産残高を一括登録できる。

CSVには、少なくとも以下の項目を含める。
- 対象年月
- 資産口座・保管場所を識別する情報
- 月末残高
- 備考

残高の記録単位が商品単位に設定されている資産口座については、
口座単位の月末資産残高として登録できないものとする。

### 5.3 商品別月末残高CSV
商品単位で残高を管理する資産口座について、
保有商品の月末評価額を一括登録できる。

CSVには、少なくとも以下の項目を含める。
- 対象年月
- 資産口座を識別する情報
- 保有商品を識別する情報
- 月末評価額
- 備考

残高の記録単位が口座単位に設定されている資産口座については、
商品別月末評価額として登録できないものとする。

### 5.4 CSVファイルの選択と読み込み
利用者は、インポート対象に応じたCSVファイルを選択できる。

システムはCSVファイルを読み込み、
登録前に内容を検証する。

対応する文字コード、区切り文字およびファイルサイズの上限は、
非機能要件または詳細設計で定義する。

### 5.5 入力内容の検証
システムは、少なくとも以下を検証する。
- 必須項目が入力されていること
- 対象年月が正しい形式であること
- 残高または評価額が数値であること
- 残高または評価額が0以上であること
- 商品単位で管理する資産口座について、対象年月時点で登録対象となる有効な保有商品がすべてCSVに含まれていること
- 商品単位で管理する有効な保有商品がすべてCSVに含まれていること
- 対象の資産口座・保管場所が登録済みであること
- 対象の保有商品が登録済みであること
- 資産口座の残高記録単位とCSVの種類が一致すること
- 同一対象年月の残高が重複していないこと
- CSVファイル内に同一データが重複していないこと

### 5.6 インポート内容のプレビュー
利用者は、データを登録する前に、
CSVから読み込んだ内容を画面上で確認できる。

プレビュー画面では、少なくとも以下を確認できる。
- 読み込み件数
- 登録可能件数
- エラー件数
- 各行の登録内容
- 各行のエラー内容

利用者が登録を確定するまで、
CSVの内容を月末残高として保存しない。

### 5.7 エラー処理
有効な資産口座・保管場所または保有商品の不足がある場合は、
不足している対象をエラーとして表示し、
CSV全体を登録しない。

エラー内容には、少なくとも以下を含める。
- CSV上の行番号
- エラーとなった項目
- エラー理由

Phase1では、正常な行だけを部分的に登録する機能は対象外とする。

### 5.8 重複データの扱い
登録済みの資産口座または保有商品について、
同一対象年月の月末残高が存在する場合は、
重複エラーとして登録しない。

Phase1では、CSVインポートによる既存データの上書き更新は行わない。

既存データを変更する場合は、
月末資産残高管理または商品別月末残高管理の編集機能を使用する。

### 5.9 登録結果の表示
インポートが完了した場合、
利用者に登録結果を表示する。

登録結果には、少なくとも以下を含める。
- 登録件数
- 対象年月
- 登録完了日時

### 5.10 CSVテンプレートの出力
利用者は、インポートに使用するCSVテンプレートをダウンロードできる。

Phase1では、以下のテンプレートを提供する。

- 月末資産残高登録用CSV
- 商品別月末残高登録用CSV

出力されたCSVには、
前月末時点の残高または評価額を初期値として設定する。

利用者は、
当月末時点の内容へ修正した上でインポートするものとする。

## 6. 資産状況・推移表示
### 6.1 現在の資産状況の表示
利用者は、最新の月末資産残高をもとに、
現在の資産状況を確認できる。

少なくとも以下を表示する。
- 総資産
- 利用可能資産
- 利用可能資産として扱わない資産
- 資産口座・保管場所ごとの残高
- 前月比
- 前月比率

総資産は、
同一対象年月における各資産口座・保管場所の残高を合計して算出する。

商品単位で残高を記録する資産口座については、
保有商品の月末評価額の合計を当該口座の残高として扱う。

口座単位の月末残高と商品単位の月末評価額を
重複して総資産へ加算しないものとする。

### 6.2 総資産推移の表示
利用者は、月末時点の総資産推移を時系列で確認できる。

表示対象は月単位とし、
各対象年月の総資産をグラフおよび一覧で表示する。

少なくとも以下を確認できる。
- 対象年月
- 総資産
- 前月との差額
- 前月比率

対象年月の一部の残高が未登録の場合は、
当該月の資産情報が未確定であることを表示する。

### 6.3 口座別資産推移の表示
利用者は、資産口座・保管場所ごとの残高推移を確認できる。

表示対象とする資産口座・保管場所を選択でき、
選択した対象の月末残高を時系列で表示する。

無効化された資産口座・保管場所についても、
過去の残高履歴が存在する場合は資産推移を確認できる。

### 6.4 商品別資産推移の表示
利用者は、商品単位で残高を記録する資産口座について、
保有商品ごとの月末評価額の推移を確認できる。

表示対象とする資産口座または保有商品を選択でき、
選択した対象の月末評価額を時系列で表示する。

無効化された保有商品についても、
過去の商品別月末残高履歴が存在する場合は資産推移を確認できる。

### 6.5 資産内訳の表示
利用者は、最新の月末資産残高について、
資産の内訳を確認できる。

少なくとも以下の単位で内訳を表示する。
- 資産口座・保管場所別
- 商品種別別
- 利用可能資産かどうか

内訳の合計額は、
同一対象年月の総資産と一致するものとする。

### 6.6 表示対象期間の指定
利用者は、資産推移の表示対象期間を指定できる。

Phase1では、少なくとも以下から選択できる。
- 直近6か月
- 直近1年
- 全期間

### 6.7 残高記録単位変更時の表示
資産口座の残高記録単位が途中で変更されている場合でも、
変更前後を含めた口座残高の推移を連続して確認できる。

各対象年月の口座残高は、
当該年月に適用されている残高記録単位に基づいて算出する。

残高記録単位の変更履歴および適用開始年月の管理方法は、
基本設計または詳細設計で定義する。

### 6.8 未登録・未確定データの表示
対象年月に必要な資産口座または保有商品の残高が不足している場合は、
当該月の資産状況を未確定として表示する。

未確定の対象年月については、
少なくとも以下を確認できる。

- 未登録の資産口座・保管場所
- 未登録の保有商品
- 未確定であることを示す警告

未確定の対象年月を総資産推移へ表示する場合は、
確定済みの対象年月と区別できるようにする。
