# プロジェクト憲章

# Life Planner

## 1. プロジェクト概要
Life Plannerは、資産管理、ライフイベント、大きな支出のシミュレーションを通して、
人生における重要な意思決定を支援するライフプランニングアプリケーションである。

本プロジェクトは実際に継続利用することを前提とした個人開発であり、
設計力・実装力・保守性への考え方を確認できる成果物として公開する。

---

## 2. ビジョン
「感覚」ではなく「データ」を基に、
将来の大きな意思決定を行える状態を実現する。

利用者が毎月の資産状況を継続的に記録し、
- 現在の資産を把握する
- 将来のライフイベントをシミュレーションする
- 根拠を持って意思決定する

ことを支援する。

---

## 3. プロジェクト目的
本プロジェクトでは次の3つを同時に達成する。

### ① 実用性
自分自身が長期間利用できる資産管理アプリケーションを開発する。

### ② 技術力向上
以下の技術・設計を実践する。
- React
- TypeScript
- Docker
- REST API
- RDB設計
- DDDを意識した設計
- ADRによる設計判断の記録
- 自動テスト

### ③ ポートフォリオ
設計から運用まで一貫して考えられるエンジニアであることを
GitHub上で説明できる成果物を作る。

---

## 4. 開発方針

### 小さく作る
一度に完成を目指さず、
動作する最小単位を継続的に完成させる。

### ドキュメント駆動
実装より先に、

- 要件定義
- ユビキタス言語
- ER図
- テーブル定義
- ADR

を整備する。

### 継続利用
デモアプリではなく、
毎月実際に資産を登録し続けられる品質を目指す。

### 保守性
実装速度よりも、

- 可読性
- 保守性
- 拡張性
- 一貫性

を優先する。

---

## 5. スコープ

### Phase1
- 資産口座管理
- 保有商品管理
- 月末資産残高管理
- 月末資産状況の確定
- 資産状況表示
- 資産推移表示
- 手取り収入管理
- 目的達成判定

### Phase2
- CSVインポート
- 資産移動履歴
- 資産増減理由管理
- 目的達成時期試算

---

## 6. 成功条件

プロジェクトは次の状態を成功とする。
- 月末資産を継続して記録できる
- 最新の確定済み資産状況を確認できる
- 資産推移を確認できる
- 目的達成可否を判定できる
- GitHubから設計意図を理解できる
- 要件・設計・実装・テストが対応付けられている

---

## 7. 設計原則
### 単一の事実源（Single Source of Truth）
同じ情報は一箇所のみで管理する。

### 履歴を重視する
過去の資産状況や判定結果を再現できる設計を目指す。

### 時間軸を明確にする
「現在」ではなく、
「対象年月時点」の情報を基準として扱う。

### 業務ルールを優先する
実装都合ではなく、
業務ルールを先に定義する。
